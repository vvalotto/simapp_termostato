Vuelco ideas, quizás desordenadas para que se trabajen con Claude Code.

Todavia no tengo muy clara las diferencias entre skills, subagents y slash comand, como herramientas que ofrece
Claude Code para asistir al desarrollo de proyectos de software.

Inicialmente creo entender un slash commands como ordenes que el usuario (en este caso el desarrollador o ingeniero), 
ejecuta desde el CLI para realizar tareas repetitivas o frecuentes en la interacción con el proyecto.
Por ejemplo eso es lo que pense al especificar el esquema de tracking dentro de este proyecto. Si bien está
implementado en el skill implement-us, no veo los comandos como parte del menu de slash commands del CLI.

Por otro lado, entiendo por lo que lei, que los subagentes son esquemas que definen comportamientos asignados a
Claude Code ante eventos o situaciones requeridos por el usuario. Estos comportmientos ademas de ejecutar operaciones,
generan artefactos dentro del proyecto, realizan análisis y toman decisiones.

Por ultimo están los skills, y aqui es donde se me confunden con los subagentes y no se cual es la diferencia.

Además de lo onceptual, no se muy bien las implementaciones de cada herramienta.
